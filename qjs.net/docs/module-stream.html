Stream

统一字节流抽象 · C# 端实现的 pipe · 引擎兜底关闭。

设计

  • 包装任意 System.IO.Stream（文件、内存、HTTP 响应体……）。
  • read / write 为异步（返回 Promise）；pipe 在 C# 端用循环实现，避免 per-chunk JS↔native 往返。
  • 所有 Stream 实例由引擎登记跟踪——即使 JS 忘记 close()，引擎销毁时也会强制释放底层句柄。

静态方法 static

Stream.memory(initialCapacity?: number): Stream

创建可读可写的内存 Stream（基于 MemoryStream），常用作 pipe 的目的地。

属性

readable: boolean
writable: boolean
closed: boolean
position: number
length: number

不可 seek 时 position / length 为 -1。

方法

read(buf: Buffer, offset?: number, length?: number): Promise<number> async

读到 buf 的 [offset, offset+length)。length 缺省/<=0 时为 buf.length-offset。返回实际读入字节数；0 表示 EOF。

write(data, offset?: number, length?: number): Promise<number> async

data 接受 Buffer / ArrayBuffer / Uint8Array / string（按 UTF-8 编码）。length 缺省/<=0 时写入剩余全部。

flush(): Promise<void> async
close(): Promise<void> async

幂等。关闭后其它方法抛 InvalidOperationException("Stream is closed")。

pipe(dest: Stream, chunkSize?: number): Promise<number> async

按 chunkSize（缺省/<=0 时 64 KiB）拷贝本流剩余字节到 dest，始终在完成时关闭 dest。返回总字节数。

readAll(maxBytes?: number): Promise<Buffer> async

一次读完剩余字节到新 Buffer。maxBytes 缺省/<=0 时不限。

生命周期与资源安全

  • 显式 close 优先：建议在 finally 中 await s.close()。
  • 引擎兜底：未关闭的 Stream 会在 engine.Dispose() 时被强制 Dispose（含底层 FileStream / HttpResponseMessage）。
  • 幂等：多次 close 安全；并发调用 close 与 read/write 时，未完成的 IO 会抛 ObjectDisposedException。

示例：HTTP 响应体直接 pipe 到文件

var resp = await fetch('https://example.com/large.bin');
var src  = resp.body();                       // 流式响应体
var dst  = fs.openWrite('large.bin');
var n    = await src.pipe(dst);              // dst 完成后自动关闭
await src.close();
console.log('downloaded', n, 'bytes');

示例：Stream.memory 作为内存 sink

var sink = Stream.memory();
await sink.write(Buffer.from('hello'));
await sink.write(' world');            // 字符串按 UTF-8
sink.length;                                // 11
await sink.close();
