[JSExport] 类

  • 类型必须是 partial sealed class，源生成器会为其生成 binder。
  • JS 端方法/属性名 = C# 名首字母小写。
  • 构造函数：当前不支持从 JS 端 new 出 [JSExport] 类——通过 C# 端 SetGlobal() 或方法返回值传入。
  • 静态成员：用 engine.SetGlobalStatic<T>("Name") 在 JS 全局暴露一个静态容器对象。
  • 事件：C# 端 EventHandler / EventHandler<T> 自动映射为 JS 端 addEventListener / removeEventListener。
  • 实现 IDisposable 的 [JSExport] 类自动登记到引擎，引擎销毁时强制释放。

方法形参 & 默认值

重要C# 方法的默认参数值不会透传到 JS 调用：JS 端没传的参数在 C# 端读为 0 / false / null / 0.0。设计 [JSExport] API 时应让“缺省即正确语义”——例如用 length <= 0 表示“读全部”，避免依赖 C# 默认参数值。

异常

C# 抛出的 Exception 会被生成器代码捕获，转为 JS 端 throw new Error(message)。JS 抛出的异常在 Eval/Execute 中以 QuickJSException 抛出。
