当前版本局限

已知限制 · 替代方案 · 未在范围内的功能。

Buffer 不支持 buf[i] 索引

QuickJS 的 exotic-class 索引拦截改造成本高，且会绕过 [JSExport] 生成器。请使用 buf.get(i) / buf.set(i, v)。

方法默认参数不会透传

JS 端没传的参数在 C# 一律读为 0 / false / null，不会取 C# 方法签名上写的默认值。设计 [JSExport] API 时应让“缺省=正确语义”，详见 类型映射。

[JSExport] 限制

  • 类必须是 partial sealed class。
  • 不支持泛型类型/方法。
  • 不支持 ref / out / in 形参。
  • 不支持继承多态——派生类需要单独标 [JSExport]，binder 按运行时类型严格匹配。
  • JS 端目前不能 new 出 [JSExport] 类；通过 C# 端 SetGlobal 或方法返回值传入。
  • 方法重载按 JS 名注册，同名重载会互相覆盖。请用唯一 JS 名或合并为单一签名（用 JSValue 形参在 C# 内分派）。
  • 数组形参元素类型必须是标量 / string / byte[] / 已注册 [JSExport] 类——不支持嵌套数组。

单线程 / 单上下文

不要并发调用同一个 QuickJSEngine。需要并发 → 多个引擎实例。

不在范围内

  • 没有 TypedArray 直接 zero-copy 视图（QuickJS 不暴露安全 API）。Buffer.toUint8Array() 必然拷贝。
  • 没有 Web Streams（ReadableStream / WritableStream）。我们用更轻量的统一 Stream 类；与 Node 的 stream.Readable 也不兼容。
  • 跨上下文传递 Stream / Buffer 行为未定义。
  • 没有 worker_threads / SharedArrayBuffer。

模块系统

支持标准 ES Module 语法（import / export / 顶层 await / 动态 import()），见 ES Modules。当前局限：

  • import.meta.url / import.meta.main 已支持；main 当前固定为 false（保留字段）。
  • 不支持 import attributes / import assert { type }。
  • 不支持远端 (https://...) 模块。
  • 内置模块默认仅以全局形式暴露；启用 QuickJSEngineOptions.BuiltinAsModule 后可通过 import { fs } from 'qjs:fs' 等使用，BuiltinAsModuleOnly 可同时移除全局。
