ES Modules（import / export）

标准 ESM 语法 · 虚拟模块 · 文件系统模块 · 自定义解析器 · 顶层 await · 动态 import。

能做什么

  • JS 端使用 import { x } from "..." / export const x = ... / export default ...。
  • 顶层 await（模块求值是异步的，由内置 EventLoop 驱动）。
  • 动态 import("...")，返回模块命名空间 Promise。
  • 循环导入（QuickJS 自带 dedupe）。

三种模块来源

所有模块解析均通过 engine.Modules（ModuleLoader）完成。解析顺序：虚拟模块 → 自定义 Resolver → 文件系统。

using var engine = new QuickJSEngine();

// 1) 虚拟模块（内存中注册源码）
engine.Modules.Register("math", "export const add = (a,b) => a+b;");

// 2) 文件系统沙箱（限定根目录）
engine.Modules.BasePath = @"D:\scripts";   // 之后允许 "./a.mjs" / "/abs/inside/sandbox"

// 3) 自定义解析器（最高优先级，仅用于在已注册之外动态生成源）
engine.Modules.Resolver = name => name == "hello" ? "export default 'hi';" : null;

执行模块代码

有两种入口：

  • engine.ExecuteModule(src, fileName?) —— 把 src 当作模块源（支持 import / export）执行；返回值为该模块的求值结果（通常 undefined，模块的输出请通过 globalThis 或 engine.Import(...) 取回）。
  • engine.Import(specifier) —— 等价于在 JS 里 const m = await import(specifier)，返回一个 .NET 字典样的对象（实际上是 JSON 化的对象字符串，因为引擎约定如此；解析它请用 System.Text.Json）。
engine.Modules.Register("util", "export const x = 42;");
var ns = engine.Import("util");     // '{"x":42}'

engine.ExecuteModule(@"
    import { add } from 'math';
    globalThis.result = add(1, 2);
");
var r = engine.GetGlobal("result");   // 3

Specifier 命名约定

形式含义
"foo" / "my-mod"裸 specifier。先查 Modules.Register，再查 Resolver。
"./a.js" / "../b.mjs"相对路径。需要先设置 Modules.BasePath。
"/abs/c.js"绝对路径，必须落在 BasePath 内（沙箱）。
"qjs:fs" / "qjs:fetch" 等内置模块，需启用 BuiltinAsModule 选项（见下方）。
用户自定 native 模块通过 Modules.RegisterNative(name, builder => ...) 注册，可使用任意名字。

不会自动补 .js / .mjs 扩展名；用户写完整文件名。

顶层 await & 动态 import

engine.Modules.Register("slow", @"
    await new Promise(r => setTimeout(r, 10));
    export const ready = true;
");

engine.ExecuteModule(@"
    const m = await import('slow');
    globalThis.ok = m.ready;
");
// engine.GetGlobal(""ok"") === true

当前限制

  • 不支持 import attributes / import assert { type: "json" }。
  • 不支持 https://、node:、data: 等 URL specifier。
  • 同一 specifier 字符串被 QuickJS 视为同一模块（dedupe），不要用相对路径绕过缓存。

模块源码大小不再受 1 MiB 限制（Phase B-1 起改为按需 js_malloc）。

原生模块（C# 注册）

通过 engine.Modules.RegisterNative(name, builder => ...) 把 C# 值/函数/类暴露成 ES Module。Specifier 由你自己取（建议用 qjs:xxx 或 app:xxx 命名空间）。

engine.Modules.RegisterNative("app:math", b =>
{
    b.Export("PI", 3.14159);
    b.ExportFunc("add", new Func<int, int, int>((a, b) => a + b));
    b.ExportDefault("hello");
});

engine.ExecuteModule(@"
    import def, { PI, add } from 'app:math';
    globalThis.r = add(1, 2);          // 3
    globalThis.p = PI;                 // 3.14159
    globalThis.d = def;                // 'hello'
");

Builder API：Export(name, value) / ExportFunc(name, Delegate) / ExportFactory(name, factory)（延迟求值）/ ExportDefault(value) / ExportDefaultFunc / ExportDefaultFactory / ExportRaw(name, JSValue)。

把内置模块当 ESM 使用

默认所有内置 API 都是 globals（fs / fetch / Buffer ...）。把 BuiltinAsModule 打开后，同一份实现也可通过 import 拉取：

var opt = new QuickJSEngineOptions {
    BuiltinAsModule = true,        // 让 qjs:fs / qjs:fetch ... 可 import
    BuiltinAsModuleOnly = false,    // true 则同时移除 globals（沙箱场景）
};
using var engine = new QuickJSEngine(opt);

engine.ExecuteModule(@"
    import { readFileSync } from 'qjs:fs';
    import { fetch } from 'qjs:fetch';
    import { Buffer } from 'qjs:buffer';
    globalThis.t = readFileSync('./README.md');
");

支持桥接的模块名：qjs:fs、qjs:fsAsync、qjs:fetch、qjs:buffer、qjs:stream、qjs:directorystream、qjs:timers、qjs:encoder、qjs:dialogs。每个模块的导出名就是对应 global 的属性名。

import.meta

每个模块都有 import.meta，目前提供两个字段：

字段类型说明
import.meta.urlstring模块 URL，按来源生成（见下表）。
import.meta.mainboolean当前为占位（始终 false），保留给后续标记入口模块用。
模块来源URL 形式
文件系统（BasePath）file:///<绝对路径，正斜杠>
虚拟模块（Register）qjs-virtual:<name>
原生模块（RegisterNative）qjs-native:<name>
自定义 Resolverqjs-resolver:<name>
engine.Modules.Register("foo", "export const u = import.meta.url;");
engine.ExecuteModule("import { u } from 'foo'; globalThis.x = u;");
// engine.GetGlobal("x") === "qjs-virtual:foo"
