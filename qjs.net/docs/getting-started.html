快速上手

从安装到运行第一段 JS，仅需 5 步。

1. 安装与引用

项目目前以源码形式集成。把 qjs.net/QuickJsNet.csproj（含 QuickJsNet.SourceGenerators）作为引用加入你的解决方案。库依赖原生 quickjs 动态库，已通过 P/Invoke 加载。

2. 创建引擎

using QuickJsNet;
using var engine = new QuickJSEngine(opt => {
    opt.FileSystem        = true;
    opt.AsyncFileSystem   = true;
    opt.Fetch             = true;
    opt.FileSystemBasePath = @"C:\sandbox"; // fs/fsAsync 根目录
});

所有模块都是按需启用，未启用的不会污染全局命名空间。Buffer 与 Stream 始终可用（被其它模块依赖）。

3. Eval vs Execute

方法异步顶层 await用途
engine.Eval(code)否否纯同步表达式；遇到 Promise 不展开
engine.Execute(code)是（在调用线程上 pump 事件循环）是所有含 await / 异步副作用的脚本
注意所有用到 fetch / fsAsync / setTimeout 等的脚本都必须用 Execute，否则 Promise 不会被解开。

4. 注入全局对象

engine.SetGlobal("appName", "qjs.net");

// 让 [JSExport] 类的所有静态成员暴露在 JS 全局
engine.SetGlobalStatic<MyMath>("MyMath");

engine.Execute("console.log(appName, MyMath.add(1, 2))");

5. 暴露 C# 类给 JS

[JSExport]
public sealed partial class Counter {
    private int _n;
    public int Value => _n;
    public int Inc(int by = 1) => _n += by;
    public async Task<int> IncAsync(int by) {
        await Task.Delay(10);
        return _n += by;
    }
}

engine.SetGlobal("counter", new Counter());
engine.Execute(@"
    counter.inc(5);
    var v = await counter.incAsync(3);
    v;     // 8
");

命名约定：方法/属性首字母在 JS 端自动小写。async Task<T> 自动映射为返回 Promise<T> 的方法。详见 类型映射。

资源释放

始终用 using 包裹 QuickJSEngine。引擎销毁时会自动关闭所有未关闭的 Stream / DirectoryStream / FetchResponse。详见 Stream 中的“生命周期”章节。
