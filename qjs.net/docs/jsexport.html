JSExport 原生代理

通过 [JSExport] 特性，让 C# 类在 JavaScript 中拥有近乎原生的调用体验。

核心概念

[JSExport] 是 QuickJsNet 最核心的特性之一。它配合 .NET 源生成器（Source Generator），在编译时为你的 C# 类生成高度优化的非反射绑定代码（Binder）。

要求
  • 类必须声明为 partial。
  • 建议声明为 sealed 以获得最佳性能。
  • 必须拥有公共构造函数（虽然当前 JS 端不能 new，但 C# 返回实例时需要）。
[JSExport]
public partial class MyModule
{
    public void Hello() => Console.WriteLine("Hello from C#");
}

支持的成员类型

源生成器会自动扫描类中的公共（public）成员并暴露给 JS：

成员类型JS 表现备注
实例方法函数自动转为 camelCase。支持同步/异步。
实例属性Getter / Setter自动转为 camelCase。支持只读/只写。
静态方法容器对象上的函数需通过 SetGlobalStatic 注册。
静态属性容器对象上的属性同上。
事件EventTarget 风格支持 addEventListener / removeEventListener。
索引器不支持建议使用 get(i) / set(i, v) 方法。

命名约定与重命名

默认情况下，C# 成员在 JS 中会自动转换为 camelCase（小驼峰）风格。

  • public void DoWork() → obj.doWork()
  • public string UserName { get; set; } → obj.userName

如果你需要自定义 JS 中的名称，可以使用 [JSName] 特性：

[JSExport]
public partial class Tools
{
    [JSName("execute_all")]
    public void Run() { }
}

排除成员

如果你不希望某个公共成员被暴露给 JS，可以使用 [JSIgnore]：

[JSIgnore]
public string InternalToken { get; set; }

注意：私有（private）和内部（internal）成员默认不会被导出。

异步支持 (Async / Promise)

如果 C# 方法返回 Task 或 Task<T>，生成器会自动将其包装为 JavaScript Promise。

// C#
public async Task<string> FetchDataAsync(string url) { ... }

// JS
const data = await obj.fetchDataAsync(url);
重要 异步任务的回调会自动调度回引擎所在的单线程 EventLoop，无需担心线程安全问题。

事件支持

类中的 event Action 或 event EventHandler 会被映射为标准的 JS 事件订阅模式。

// C#
public event Action<string>? MessageReceived;

// JS
obj.addEventListener('messageReceived', (msg) => {
    console.log('Got:', msg);
});

生命周期与 IDisposable

如果 [JSExport] 类实现了 IDisposable 接口：

  1. 自动注册：该实例会被 QuickJsNet 运行时自动追踪。
  2. 强制释放：当 QuickJSEngine 被销毁（Dispose）时，所有由该引擎创建且尚未释放的 [JSExport] 实例都会被强制调用 Dispose()。

这确保了即使 JS 端没有及时触发垃圾回收，原生资源（如文件句柄、网络连接）也能被正确关闭。

如何注入到 JS

创建好 [JSExport] 类后，你可以通过以下方式将其暴露给 JS：

1. 注入实例

var myObj = new MyModule();
engine.SetGlobal("my", myObj);
// JS: my.hello();

2. 注入静态容器

这常用于注入单例工具类或工厂类：

engine.SetGlobalStatic<StaticMath>("MathUtils");
// JS: MathUtils.square(5);

局限性

  • 不能在 JS 端 new：目前不支持 new MyClass()。对象必须由 C# 创建并传入。
  • 不支持泛型类：[JSExport] 不能标记在 class MyBox<T> 上。
  • 不支持继承多态：如果 A 继承 B，且都标了 [JSExport]，JS 端只会看到实例真实类型所拥有的成员。
