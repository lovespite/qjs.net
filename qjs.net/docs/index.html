QuickJsNet 文档

将 QuickJS 嵌入 .NET 的轻量绑定库 —— 原生 [JSExport] 代理、流式 IO、Promise/EventLoop 完整调度。

这是什么

QuickJsNet 把 QuickJS 嵌入 .NET 进程，提供两类对接：

  • 原生对象代理：用 [JSExport] 标注一个 C# 类，源生成器为它在 JS 端创建一个真正的 class（属性、方法、静态成员、async 方法→Promise、事件→addEventListener），无需手写 JS 胶水代码。
  • 内置模块：fs / fsAsync / fetch / 计时器 / TextEncoder / TextDecoder / Buffer / Stream / DirectoryStream，按需启用。

主要特性

  • 原生 [JSExport] 代理（无 JS 胶水）
  • 统一的字节流 Stream，C# 端实现 pipe 避免 per-chunk 跨语言往返
  • Node 风格 Buffer（Memory<byte> 背书；slice 共享内存）
  • 异步文件系统 / 流式文件读写 / 流式目录枚举 / fetch 响应体流
  • Promise ↔ Task 互通，engine.Execute() 支持顶层 await
  • 资源生命周期兜底：所有 IDisposable 资源在引擎销毁时强制释放
  • Buffer 自动作为 byte[] 形参可接受（透过 BufferUnwrapHook）
  • 标准 ES Module：import / export / 顶层 await / 动态 import() / import.meta.url / 原生模块（RegisterNative）/ 内置模块 ESM 化（qjs:fs 等）

30 秒上手

using var engine = new QuickJSEngine();
var n = engine.Execute("1 + 2 * 3");   // 同步 / 顶层 await 都行
Console.WriteLine(n);                          // → 7

var json = engine.Execute(@"
    var r = await fetch('https://api.github.com');
    (await r.json()).current_user_url;
");
